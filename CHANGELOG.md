# 0.4.1

## Features
 - `hunim server` now rebuilds smartly: instead of rebuilding the whole site whenever anything under `src/` changes, it tracks every build input and rebuilds only what a change affects. An edited page rebuilds just itself; an edited template, component, or exec script rebuilds exactly the pages that (transitively) use it; an edited static asset is re-copied with no page rebuilds; and a change in a feed directory rebuilds that feed (posts, post list, and RSS). Templates, components, and `hunim.toml` are now watched too — previously only `src/` was. Adding or deleting files, or editing `hunim.toml`, still triggers a full rebuild, and `sitemap.xml` refreshes only on full rebuilds
 - Added `hunim dag`, which serves an interactive diagram of the site's structure at `http://127.0.0.1:8081`. Every page, template, component, and exec script is drawn as a node in a DAG, with edges for the `template` frontmatter key (including implicit feed templates), `{{ component }}` invocations, and `{{ exec }}` tags. Hovering a node traces its full dependency chain (click to pin), and a template that is referenced but doesn't exist is flagged as missing. Tags inside code samples are ignored, matching the build, and refreshing the browser rescans the site — no build required
